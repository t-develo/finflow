# /build-fix — Build Error Resolution

ビルドエラー・コンパイルエラーを段階的に修正する。

## 実行手順

### 1. ビルド実行・エラー取得
```bash
dotnet build 2>&1 | tee /tmp/build-errors.txt
```

### 2. エラーを整理
- ファイル別にグループ化
- 依存順序で並び替え（基盤のエラーから先に修正）

### 3. 1エラーずつ修正
```
エラーを確認 → 原因を特定 → 最小修正を適用 → 再ビルド → 次のエラーへ
```

### 4. テスト確認
```bash
dotnet test
```

## よくあるエラーと対処

| エラー | 原因 | 対処 |
|--------|------|------|
| `CS0246: 型が見つからない` | using 文の不足 | 適切な using を追加 |
| `CS1061: メンバーが存在しない` | インターフェース不一致 | インターフェース定義を確認 |
| `EF Pending migrations` | マイグレーション未適用 | `dotnet ef database update` |
| `Nullable: 参照型がnullの可能性` | null チェック不足 | null チェックまたは `!` を追加 |
| `DI: サービスが登録されていない` | DI登録漏れ | `Program.cs` に `AddScoped` を追加 |

## 安全原則

- **1エラーずつ修正**（まとめて変更しない）
- **修正がさらにエラーを生む場合は停止してユーザーに相談**
- アーキテクチャ変更が必要な場合は `/plan` から再開

## 終了後

修正完了したら以下を報告:
1. 解決したエラー一覧
2. 残っているエラー（ある場合）
3. 推奨される次のアクション
