# FinFlow 報告・指示フォーマット集【全ロール共通】

PL・SE・レビュアーが使う標準フォーマット。コピーして使う。

---

## PLからSEへのタスク指示

```markdown
## タスク: [タスクID] [タスク名]

### 目的
[このタスクがなぜ必要か - Whatだけでなく必ずWhyを伝える]

### 完了条件（Definition of Done）
- [ ] 条件1
- [ ] 条件2
- [ ] テストが全件パスしている
- [ ] レビュアーのAPPROVEを得ている

### 技術的な補足
[必要に応じて: 実装方針のヒント、参照すべきファイル等]

### 依存・制約
[依存タスク、注意事項、スコープ境界]

### 期限
[日付]
```

---

## SEからPLへのタスク完了報告

```markdown
## 完了報告: [タスクID]

### 実装サマリ
- [変更内容の箇条書き]

### 作成・変更ファイル
- src/FinFlow.Api/Controllers/ExpensesController.cs
- src/FinFlow.Infrastructure/Services/ExpenseService.cs
- tests/FinFlow.Tests/ExpenseServiceTests.cs

### テスト結果
- テスト件数: X件
- 全件パス: Yes / No
- カバレッジ: 概算

### 注意事項・申し送り
[他SEへの影響、既知の制約、次タスクへの引き継ぎ事項]
```

**SE-2の集計タスクの場合は追加で:**

```markdown
### 集計精度の確認
- 手計算との照合結果: [期待値との一致を確認済み]
```

**SE-3のフロントエンドタスクの場合は追加で:**

```markdown
### 動作確認チェックリスト
- [ ] 正常系の操作が動作する
- [ ] バリデーションエラーが表示される
- [ ] ブラウザバック/フォワードが正常に動作する
- [ ] ローディング状態が表示される
- [ ] エラー時のメッセージが表示される

### 確認済みブラウザ
- Chrome xx
```

## SEからPLへのブロッカー報告

**即座に**報告する。「様子を見る」は禁止。

```markdown
## ブロッカー報告: [タスクID]

### 何が起きているか
[具体的な症状・エラーメッセージ]

### 試したこと
- [試した手段1]
- [試した手段2]

### 必要な支援
[PLに何をしてほしいか: 仕様確認？技術サポート？スコープ調整？]

### 影響
[このブロッカーが解消されないと、どのタスクがブロックされるか]
```

---

## PLからレビュアーへのレビュー依頼

```markdown
## レビュー依頼

### 対象コード
- 実装者: [SE-1/SE-2/SE-3]
- タスク: [タスクID] [タスク名]
- 対象ファイル:
  - src/...
  - tests/...

### 特に注意して見てほしい点
[セキュリティ観点？集計精度？アーキテクチャ設計？]

### 背景・経緯
[必要に応じて: 仕様変更があった場合など]
```

## PL スプリント進捗レポート

```markdown

### 全体進捗
- 計画タスク数: X
- 完了タスク数: Y
- SPI（スケジュール効率指数）: Y/X

### 各SE状況
| SE | 担当タスク | 状況 | ブロッカー |
|----|-----------|------|-----------|
| SE-1 | [タスク名] | 完了/進行中/ブロック | - |
| SE-2 | [タスク名] | 完了/進行中/ブロック | - |
| SE-3 | [タスク名] | 完了/進行中/ブロック | - |

1. ...
2. ...
```

```markdown
### 良い点
- [具体的に良かった点]

### Must Fix（件数: X件）
1. **[ファイル:行番号]** [問題タイトル]
   - 問題: ...
   - 理由: ...
   - 改善案: ...

### Should Fix（件数: X件）
1. **[ファイル:行番号]** [問題タイトル]
   - 改善案: ...

### Nit
1. **[ファイル:行番号]** [簡潔な指摘]

### 学びの共有
[チーム全体に周知すべき知見]
```
