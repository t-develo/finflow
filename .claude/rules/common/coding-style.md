# Coding Style — FinFlow

## 基本原則

**イミュータビリティ（必須）:** 新しいオブジェクトを生成し、既存オブジェクトを直接変更しないこと。

**ファイルサイズ:** 200〜400行が目安、最大800行。機能・ドメイン単位でファイルを分割する。

**エラーハンドリング:** すべてのレイヤーで明示的に処理する。エラーをサイレントに握りつぶさない。

**入力バリデーション:** システム境界（ユーザー入力・外部API）で必ずバリデーションを行う。

## C# 規約

- `PascalCase`: publicメンバー、クラス名、メソッド名
- `camelCase`: privateフィールド（`_`プレフィックスは不要）
- 非同期メソッドには `Async` サフィックスを付ける（例: `GetExpensesAsync`）
- **金額フィールドには必ず `decimal` 型を使用**。`float`/`double` は禁止
- インターフェース名は `I` プレフィックス（例: `ICsvParser`）
- `var` は型が明示的に分かる場合のみ使用可

## JavaScript 規約

- ファイル名: `kebab-case.js`
- クラス名: `PascalCase`、変数・メソッド: `camelCase`
- Web Components は `ff-` プレフィックス（例: `<ff-expense-form>`）
- CSS クラス: BEM記法（例: `.expense-form__input--error`）
- ES2020+ 機能を使用可（ビルドステップなし）

- [ ] ファイルは800行以内
- [ ] ネストは最大4段
- [ ] エラーハンドリングが適切
- [ ] ハードコードされた値がない
- [ ] 金額計算に `decimal` を使用（C#）
