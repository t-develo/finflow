## 担当範囲

SE-Cは以下のUI/画面を担当します:

1. **SPA基盤** - ルーティング、レイアウト、共通コンポーネント
2. **認証画面** - ログイン・ユーザー登録
3. **支出管理画面** - 入力フォーム・一覧表示
4. **ダッシュボード** - グラフ・サマリ表示（Sprint 2）
5. **CSV取込画面** - ファイルアップロード・プレビュー（Sprint 2）
6. **サブスク管理画面** - 一覧・登録/編集フォーム（Sprint 2）
7. **カテゴリ管理画面** - 一覧・追加/編集/削除（Sprint 2）

## 共通ルール

### 技術スタック
- **JavaScript:** バニラJS（ES2020+）、フレームワーク不使用
- **コンポーネント:** Web Components（Custom Elements + Shadow DOM）
- **ルーティング:** Navigo（軽量SPAルーター）または自前実装
- **グラフ:** Chart.js（Sprint 2で使用）
- **CSS:** カスタムプロパティ（CSS変数）で統一的なデザイントークンを管理
- **ビルド:** ビルドツール不使用（直接ブラウザで動作するES Modules）

### コーディング規約
- ファイル名: kebab-case（例: `expense-form.js`）
- クラス名: PascalCase（例: `ExpenseForm`）
- メソッド名/変数名: camelCase
- Web Components のタグ名: `ff-` プレフィックス（例: `<ff-expense-form>`）
- CSS クラス名: BEM記法（例: `.expense-form__input--error`）

### API通信ルール
- 共通のfetchラッパーを使用（`src/frontend/js/utils/api-client.js`）
- JWT をlocalStorageに保存し、全リクエストの `Authorization` ヘッダーに付与
- APIエラーはユーザーに分かりやすいメッセージで表示
- Sprint 1ではモックAPI（`src/frontend/js/mocks/`）を使用可能。Sprint 2で実API接続に切替

### 報告ルール
- 各タスク完了時にPLへ完了報告（スクリーンショット + 動作説明）
- ブロッカー発生時は即座にPLへ報告
- UI仕様の判断に迷った場合はPLへ相談

**概要:**
SPAの骨格となるルーティング、レイアウト、共通CSSを構築する。

**実装内容:**

1. **ルーティング設定:**
   - `/` - ダッシュボード（Sprint 2で実装、Sprint 1では仮画面）
   - `/login` - ログイン画面
   - `/register` - ユーザー登録画面
   - `/expenses` - 支出一覧

   - `/expenses/import` - CSV取込（Sprint 2）
   - `/subscriptions` - サブスク管理（Sprint 2）
   - `/categories` - カテゴリ管理（Sprint 2）
   - 未認証時は `/login` にリダイレクト

2. **レイアウト構成:**
   ```
   +-------------------------------------------+
   |  ヘッダー（ロゴ、ユーザー名、ログアウト）     |
   +--------+----------------------------------+
   | サイド  |                                  |
   | バー    |       メインコンテンツエリア         |
   | （ナビ） |                                  |
   |        |                                  |
   +--------+----------------------------------+
   ```

3. **共通CSS:**
   - CSS変数定義（色、フォント、スペーシング、ボーダー）
   - リセットCSS
   - レイアウト用ユーティリティクラス
   - フォーム要素の共通スタイル（input, button, select）
   - テーブル共通スタイル

4. **API通信ラッパー:**
   ```javascript
   // api-client.js
   class ApiClient {
     async get(path, params) { ... }
     async post(path, body) { ... }
     async put(path, body) { ... }
     async delete(path) { ... }
   }
   ```
   - ベースURL設定
   - JWT自動付与
   - エラーハンドリング（401→ログインリダイレクト、500→エラー表示）
   - レスポンスのJSON自動パース

**完了条件:**
- [ ] ページ間遷移が正常動作（ブラウザバック含む）
- [ ] ヘッダー・サイドバー・メインエリアのレイアウトが表示される
- [ ] 共通CSSが適用されている
- [ ] API通信ラッパーが使用可能な状態

### S1-C-002: ログイン・ユーザー登録画面

| 項目 | 内容 |
|------|------|
| **優先度** | 高 |
| **工数** | 1日 |
| **依存** | S1-C-001（SPA基盤）、S0-PL-005（共通基盤 - JWT認証） |

**概要:**
ユーザー認証のためのログイン画面と新規登録画面を実装する。

**ログイン画面（`/login`）:**

```
+----------------------------------+
|          FinFlow ロゴ             |
|                                  |
|   メールアドレス                   |
|   [________________________]     |
|                                  |
|   パスワード                      |
|   [________________________]     |
|                                  |
|   [    ログイン    ]              |
|                                  |
|   アカウントをお持ちでない方は      |
|   新規登録                        |
+----------------------------------+
```

**API連携:**
```
POST /api/auth/login
Body: { "email": "...", "password": "..." }
Response: { "token": "jwt...", "user": { "id": 1, "name": "..." } }
```

**新規登録画面（`/register`）:**
- ユーザー名、メールアドレス、パスワード、パスワード確認
- バリデーション: メール形式チェック、パスワード8文字以上、確認一致

**API連携:**
```
POST /api/auth/register
Body: { "name": "...", "email": "...", "password": "...", "confirmPassword": "..." }
```

**制約:**
- JWT をlocalStorageに保存
- ログイン成功後はダッシュボード（`/`）にリダイレクト
- 認証状態はページリロード後も維持
- ログアウトボタン（ヘッダー内）でJWT削除 + `/login` にリダイレクト

**完了条件:**
- [ ] ログインフォームが表示され、入力・送信が動作
- [ ] 新規登録フォームが表示され、入力・送信が動作
- [ ] バリデーションエラーがフォーム上に表示される
- [ ] ログイン成功後にトークンが保存されリダイレクトされる
- [ ] ログアウトが動作する

### S1-C-003: 支出入力フォーム

| 項目 | 内容 |
|------|------|
| **優先度** | 高 |
| **工数** | 1.5日 |
| **依存** | S1-C-001（SPA基盤） |

**概要:**
支出を手動入力するためのフォーム画面。

**画面（`/expenses/new`）:**

```
+----------------------------------+
|  支出を追加                       |
|                                  |
|  金額 *                           |
|  [________________________] 円   |
|                                  |
|  カテゴリ *                       |
|  [▼ 食費              ]         |
|                                  |
|  日付 *                           |
|  [2026-03-08          ]          |
|                                  |
|  説明 *                           |
|  [________________________]      |
|                                  |
|  メモ                             |
|  [________________________]      |
|                                  |
|  [キャンセル]  [保存する]         |
+----------------------------------+
```

**バリデーション（フロント側）:**
- 金額: 必須、正の数値
- カテゴリ: 必須（ドロップダウン選択）
- 日付: 必須、デフォルトは今日の日付
- 説明: 必須、1文字以上

**API連携:**
```
POST /api/expenses
GET  /api/categories  ← ドロップダウン用
```

**制約:**
- 保存成功後は支出一覧（`/expenses`）にリダイレクト
- 保存成功時にトースト通知「支出を追加しました」
- 編集モード対応（`/expenses/{id}/edit` でフォームに既存データをプリフィル）
  - 編集時は `PUT /api/expenses/{id}` を使用
- Sprint 1ではモックAPI使用可

**完了条件:**
- [ ] フォームが表示され、全フィールドに入力可能
- [ ] バリデーションエラーがフィールド横に表示される
- [ ] カテゴリドロップダウンにAPIから取得したカテゴリが表示される
- [ ] 保存ボタンでAPIにPOSTされる
- [ ] 編集モードで既存データがフォームにプリフィルされる

### S1-C-004: 支出一覧画面

| 項目 | 内容 |
|------|------|
| **優先度** | 高 |
| **工数** | 1日 |
| **依存** | S1-C-003（支出入力フォーム） |

**概要:**
登録済みの支出をテーブル形式で一覧表示し、フィルタ・編集・削除ができる画面。

**画面（`/expenses`）:**

```
+---------------------------------------------------+
|  支出一覧                    [+ 支出を追加]          |
|                                                     |
|  期間: [2026-03 ▼]  カテゴリ: [全て ▼]              |
|                                                     |
|  日付       | カテゴリ | 説明           | 金額    | 操作  |
|  -----------|---------|----------------|---------|------|
|  2026/03/08 | 食費    | コンビニ 昼食   | ¥1,500  | ✏️ 🗑 |
|  2026/03/07 | 交通費  | 電車 定期券    | ¥12,000 | ✏️ 🗑 |
|  2026/03/07 | 日用品  | ドラッグストア  | ¥2,300  | ✏️ 🗑 |
|                                                     |
|  合計: ¥15,800  (3件)                                |
|                                                     |
|  [< 前へ]  1 / 5  [次へ >]                           |
+---------------------------------------------------+
```

**機能:**
- **フィルタ:** 年月（デフォルト当月）、カテゴリ（ドロップダウン）
- **ソート:** 日付降順（デフォルト）
- **ページング:** 1ページ20件
- **操作:**
  - 編集ボタン → `/expenses/{id}/edit` に遷移
  - 削除ボタン → 確認ダイアログ後に `DELETE /api/expenses/{id}`
- **合計表示:** 表示中のデータの合計金額と件数

**API連携:**
```
GET /api/expenses?year=2026&month=3&categoryId=1&page=1&pageSize=20
```

**制約:**
- 金額はカンマ区切り + 円記号で表示（例: ¥1,500）
- 削除後はリスト更新（削除した行を除去）
- データなし時は「この期間の支出はありません」メッセージ表示
- Sprint 1ではモックAPI使用可

**完了条件:**
- [ ] テーブルに支出データが表示される
- [ ] 年月・カテゴリフィルタが動作する
- [ ] 編集ボタンで編集画面に遷移する
- [ ] 削除ボタンで確認→削除が動作する
- [ ] 合計金額・件数が正しく表示される
